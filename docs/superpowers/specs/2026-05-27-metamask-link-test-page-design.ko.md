# MetaMask 연결 테스트 페이지 설계

## 목표

데스크톱 Chrome의 MetaMask 확장 프로그램 연결과 모바일 외부 브라우저에서 WalletConnect/Reown AppKit을 통한 MetaMask Android/iOS 앱 연결을 모두 지원하는 프론트엔드 전용 지갑 테스트 페이지를 만든다.

이 페이지의 목적은 실사용 QA이다. 지갑을 연결하고, 상태를 확인하고, 일반적인 지갑 기능을 실행하며, 백엔드 없이 모바일 세션 문제가 간헐적으로 발생하는 원인을 진단할 수 있게 한다.

## 기술 접근

Vite, React, TypeScript, Wagmi, Viem, Reown AppKit을 사용한다.

- Vite는 프로젝트를 가볍고 로컬에서 실행하기 쉽게 유지한다.
- React는 테스트 화면을 패널과 재사용 가능한 컨트롤 단위로 구성하기에 적합하다.
- Wagmi와 Viem은 이더리움 계정, 체인, 트랜잭션, 서명 API를 안정적으로 제공한다.
- Reown AppKit은 WalletConnect 기반 모바일 세션 연결과 지갑 선택 UI를 제공한다.

커스텀 백엔드는 포함하지 않는다. WalletConnect 세션 통신은 브라우저 앱, WalletConnect/Reown 릴레이, MetaMask 사이에서 이루어진다. 로컬 앱은 UI 상태, 지갑 요청, 설정, 브라우저 측 진단만 담당한다.

## 기능

첫 번째 버전은 다음을 포함한다.

- 지갑 연결 패널
  - 데스크톱 MetaMask 확장 프로그램 연결.
  - Reown AppKit을 통한 모바일 WalletConnect 연결.
  - 연결된 계정, 커넥터 이름, 체인 ID, 네트워크 이름 표시.
- 진단 패널
  - 현재 지갑 상태.
  - 마지막 액션, 요청 의도, 결과, 에러.
  - user agent 기반 브라우저/디바이스 힌트.
  - disconnect와 로컬 지갑 상태 초기화 같은 세션 정리 액션.
- 지갑 기능 테스트
  - 계정 및 잔액 조회.
  - 네이티브 코인 전송.
  - ERC-20 토큰 전송.
  - 네트워크 변경 및 체인 추가 흐름.
  - `personal_sign`.
  - EIP-712 typed data 서명.

## 안전성

페이지는 테스트에 안전한 기본값을 사용해야 한다.

- 기본값은 테스트넷 중심으로 둔다.
- 전송에는 받는 주소와 수량을 사용자가 직접 입력하게 한다.
- 전송 또는 서명 전 활성 체인을 보여준다.
- 지갑 거절과 체인 불일치 에러를 명확히 표시한다.
- 세션 정리처럼 파괴적인 액션은 명시적인 버튼으로만 실행한다.

## 설정

사용자 또는 배포 환경마다 달라지는 값은 프론트엔드 환경 변수로 관리한다.

- `VITE_REOWN_PROJECT_ID`
- 지원 체인을 위한 선택적 RPC URL

네트워크와 토큰 예시는 소스 설정 파일에 둔다. 나중에 필요하면 백엔드 `/config` 엔드포인트에서 내려주는 구조로 바꿀 수 있다.

## 향후 백엔드 확장

모바일 세션 문제를 장기적으로 추적해야 한다면, 지갑 연결 구조를 바꾸지 않고 작은 백엔드를 나중에 추가할 수 있다.

가능한 백엔드 역할:

- 네트워크/토큰 설정 제공.
- 액션 로그와 에러 리포트 저장.
- 디바이스, 브라우저, 체인, 커넥터별 결과 비교.

백엔드는 지갑 세션을 릴레이하거나 지갑 액션을 승인하지 않는다.

## 테스트

다음을 검증한다.

- 앱이 성공적으로 빌드된다.
- 데스크톱 Chrome의 MetaMask 확장 프로그램으로 연결하고 조회/서명 기능을 실행할 수 있다.
- 모바일 외부 브라우저에서 WalletConnect/Reown 흐름을 통해 MetaMask 앱을 열 수 있다.
- disconnect와 로컬 상태 초기화가 오래된 세션 복구를 쉽게 만든다.
- 요청 거절, project ID 누락, 미지원 체인, RPC 요청 실패 에러가 화면에 표시된다.
